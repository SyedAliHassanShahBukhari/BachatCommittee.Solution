# Designer Requirements Document (UI/UX)
## Bachat Committee Management System (Web Admin + Mobile App)

**Document ID:** BCS-DES-001  
**Version:** 2.0 (Combined & Final)  
**Date:** 2025-12-29  
**Audience:** UI/UX Designers, Product Designers  
**Purpose:** This is the **single, complete, authoritative Designer Document**, combining all prior Designer Requirements into one consolidated reference.  
**Guarantee:** A designer can design the **entire system end-to-end** without additional requirement-gathering sessions.

---

## 0. Objective of This Document

This document defines **everything a designer needs** to design the Bachat Committee system:

- Dashboards for **all user roles**
- Navigation & access-controlled menus
- Screen inventory (Web + Mobile)
- Forms, tables, lists, charts
- Component behaviors and states
- Realtime UI behavior
- Sample data for realistic mockups
- Edge cases and failure states
- Accessibility and usability rules

**Expected Designer Output**
- High-fidelity Figma (or equivalent)
- Responsive layouts
- Component library
- Interactive prototypes for all core flows

## 1. User Roles & Responsibilities

### 1.1 Roles Overview

| Role | Platform | Core Responsibility |
|---|---|---|
| Super Admin | Web | Governance, oversight, dispute resolution |
| Pool Admin (optional) | Web | Pool-level operations |
| Account Member | Mobile (Primary), Web (Optional) | Payments, confirmations |
| Receiver-of-the-Month | Mobile | Incoming payment tracking |
| Guest Member | None | Represented visually only |

## 3. Dashboards (CRITICAL)

---

## 3.1 Super Admin Global Dashboard (Web)

### Purpose
Immediate operational visibility across **all pools**

### Sections

#### A. Summary Cards
- Total Pools
- Active Pools
- Total Members (Account / Guest)
- Outstanding Payments (current cycle)
- Open Disputes

#### B. Pool Health Table
Columns:
- Pool Name
- Status
- Current Cycle
- % Payments Completed
- Late Members Count
- Action (View Pool)

Row colors:
- Green (>90%)
- Yellow (60–90%)
- Red (<60%)

#### C. Payments Status Chart
- Donut or stacked bar:
  - Paid
  - Pending
  - Late
  - Disputed

#### D. Reliability Distribution Chart
- Pie chart:
  - Green
  - Orange
  - Yellow
  - Red

#### E. Activity Feed
- “Admin confirmed payment on behalf of Guest X”
- “Split payout configured for Pool A – Cycle 4”
- “Schedule regenerated by Admin”

## 13. Edge Cases Designers Must Handle

- Guest member confirmations (admin-only)
- Member in multiple pools
- Split payout with 3+ receivers
- Reminder snoozed but still unpaid
- Offline/syncing state
- Conflicting confirmations → Disputed highlight

---

## 14. Accessibility & Usability Rules

- Large tap targets (mobile)
- Clear labels (no icon-only actions)
- Plain language error messages
- Avoid technical jargon for members
- WCAG-friendly contrast
